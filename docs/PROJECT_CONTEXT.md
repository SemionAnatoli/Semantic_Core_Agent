# PROJECT_CONTEXT.md

## Название
**Агент коммерческого семантического ядра и кластеризации**

## Описание
Мульти-агентная система, которая собирает поисковые запросы из множества источников,
чистит их, определяет интент, кластеризует под страницы и выдаёт готовый план действий:
создать / обновить / FAQ / блог / не брать. Результат — Google Таблица с приоритетами.

## Цели проекта (что считается успехом)
- На вход: сайт + конкуренты + seed-ключи → на выходе: готовый план SEO-работ
- Агент принимает решения сам, не просто генерирует ключи
- Результат читаем не-техническим пользователем (SEO-специалист, клиент)
- Управление через Telegram-бот или простой веб-интерфейс

## Целевая аудитория
- SEO-специалисты
- Digital-агентства
- Владельцы интернет-магазинов

## Ключевые ограничения
- Инструменты RU-рынка: Wordstat, Keys.so, SpyWords, Яндекс.Вебмастер
- Инструменты GL-рынка: Google Search Console, Google Ads API
- Выгрузка в Google Sheets (удобно для клиентов)
- Управление: Telegram-бот (основной вариант) или веб-UI

## Стек технологий
| Технология | Роль |
|-----------|------|
| Python 3.11+ | Основной язык |
| LangGraph | Оркестровка мульти-агентного пайплайна |
| Claude API (Anthropic) | LLM: intent, чистка, expansion, clustering |
| Keys.so / SpyWords API | Данные конкурентов и ключевых слов |
| Google Search Console API | Существующие запросы сайта |
| Яндекс.Вебмастер API | Яндекс-трафик и запросы |
| pytrends | Google Trends |
| gspread | Выгрузка в Google Sheets |
| python-telegram-bot | Telegram-бот управления |
| FastAPI + Jinja2 | Опциональный веб-интерфейс |

## Архитектура верхнего уровня

```
INPUT: {site_url, business_description, seed_keywords, geo, competitors[]}
                          │
              ┌───────────▼───────────┐
              │    RESEARCH AGENT     │  ← GSC + Яндекс.Вебмастер + Wordstat
              │  (сбор данных)        │  ← Keys.so / SpyWords / конкуренты
              └───────────┬───────────┘
                          │
              ┌───────────▼───────────┐
              │   EXPANSION AGENT     │  ← модификаторы, long-tail, вопросы
              │  (расширение ядра)    │  ← вертикаль + горизонталь
              └───────────┬───────────┘
                          │
              ┌───────────▼───────────┐
              │   CLEANING AGENT      │  ← дубли, мусор, нерелевант
              │  (чистка)             │  ← стоп-слова, бренды, риски
              └───────────┬───────────┘
                          │
              ┌───────────▼───────────┐
              │    INTENT AGENT       │  ← LLM классификация
              │  (определение интента)│  ← commercial/info/local/nav/transact
              └───────────┬───────────┘
                          │
              ┌───────────▼───────────┐
              │  CLUSTERING AGENT     │  ← группировка под страницы
              │  (кластеризация)      │  ← SERP-based + entity-based
              └───────────┬───────────┘
                          │
              ┌───────────▼───────────┐
              │ PRIORITIZATION AGENT  │  ← scoring-формула
              │  (приоритизация)      │  ← volume × value × opportunity
              └───────────┬───────────┘
                          │
              ┌───────────▼───────────┐
              │   MAPPING AGENT       │  ← кластер → страница → действие
              │  (маппинг)            │  ← создать/обновить/блог/FAQ/не брать
              └───────────┬───────────┘
                          │
              ┌───────────▼───────────┐
              │   OUTPUT AGENT        │  ← Google Sheets
              │  (выгрузка)           │  ← JSON + таблица
              └───────────┬───────────┘
                          │
                   Google Таблица
                   (готовый план)
```

## Глоссарий
| Термин | Определение |
|--------|-------------|
| Seed keywords | Стартовые ключи, с которых начинается расширение |
| Кластер | Группа запросов с одним интентом → одна страница |
| Интент | Цель пользователя: купить / узнать / найти место / сравнить |
| Scoring | Формула приоритета кластера по 8 факторам |
| Маппинг | Привязка кластера к конкретной странице + действию |
| Gap-анализ | Сравнение: что ищут vs что есть на сайте |
| Каннибализация | Когда две страницы конкурируют за один запрос |
| Landing map | Карта посадочных: кластер → URL → действие |
