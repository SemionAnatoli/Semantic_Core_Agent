# Semantic Core Agent

Мульти-агентная система сбора и обработки семантического ядра для SEO.

Принимает на вход сайт, описание бизнеса и стартовые ключевые слова.  
Возвращает готовый план работ: какие страницы создать, какие обновить, в каком порядке.

---

## Что делает система

```
Вход: сайт + ключи + конкуренты
          ↓
   9 специализированных агентов
          ↓
Выход: HTML-отчёт + CSV-таблица
```

Каждый агент выполняет одну конкретную задачу и передаёт результат следующему.

---

## Быстрый старт

### Одним кликом

Дважды кликнуть на файл `run.bat` — откроется консоль, запустится анализ, браузер откроет готовый отчёт.

### Из консоли

```bash
python test_pipeline.py
```

### Настройка входных данных

Открыть `test_pipeline.py` и изменить блок `TEST_INPUT`:

```python
TEST_INPUT = {
    "site_url":             "ваш-сайт.ru",
    "business_description": "чем занимается бизнес",
    "seed_keywords":        ["ключ 1", "ключ 2", "ключ 3"],
    "geo":                  "Москва",
    "target_pages":         ["/catalog/", "/blog/"],
    "competitors":          ["конкурент1.ru", "конкурент2.ru"],
    "commercial_priority":  {"категория1": 5, "категория2": 3},
}
```

## Агенты

| Агент | Что делает |
|-------|-----------|
| **Research** | Собирает запросы из GSC, Яндекс.Вебмастера, анализирует конкурентов |
| **Expansion** | Расширяет ядро через модификаторы (гео, цена, характеристики) + генерация через LLM |
| **Cleaning** | Удаляет дубли, мусорные запросы, однословники |
| **Intent** | Классифицирует каждый запрос: commercial / informational / local / comparison / problem_based |
| **Clustering** | Группирует запросы в кластеры — 1 кластер = 1 страница |
| **Prioritization** | Считает приоритет по формуле из 8 факторов |
| **Mapping** | Привязывает кластер к странице: создать / обновить / статья / FAQ |
| **Cannibalization** | Находит конфликты между страницами (дублирование запросов, одинаковые URL) |
| **Brief** | Генерирует Title, H1, Description и структуру страницы для каждого кластера |

---

## Формула приоритета

```
Priority Score =
  0.25 × частотность запросов
+ 0.20 × ценность для бизнеса
+ 0.20 × реальность выхода в топ
+ 0.15 × соответствие интента
+ 0.10 × рост тренда
+ 0.10 × пробел в контенте
- 0.20 × сложность продвижения
- 0.15 × риск каннибализации
```

Итог — число от 0 до 100. Кластеры выше 70 — в первую очередь.

## Результат

После запуска в `data/output/` появляются:

| Файл | Что внутри |
|------|-----------|
| `report.html` | Интерактивный дашборд с графиками — открыть в браузере |
| `test_result.csv` | Таблица кластеров — открыть в Excel или Google Sheets |

В таблице для каждого кластера: главный ключ, все запросы, интент, рекомендуемый URL, действие (создать / обновить / статья), приоритет, причина.

## Особенности

**Работает без API-ключей.** Полный цикл на моках — можно сразу смотреть как выглядит результат.

**Два режима кластеризации:** простой (правила, без API) и умный (через Claude — понимает семантику, а не просто слова).

**Детектор каннибализации** — система сама находит конфликты: когда два кластера претендуют на один URL или когда один запрос попал в несколько кластеров.

**ТЗ для копирайтера** — для каждого кластера автоматически генерируется Title, H1, Description, рекомендуемая структура страницы и список LSI-слов.

**Telegram-бот** — можно запускать анализ прямо из мессенджера командой `/analyze`.

**Выгрузка в Google Sheets** — если настроен API, таблица улетает в нужный spreadsheet автоматически.

## API-ключи (опционально)

Скопировать `.env.example` в `.env` и заполнить:

```env
ANTHROPIC_API_KEY=sk-ant-...        # умная кластеризация и генерация ТЗ
GOOGLE_APPLICATION_CREDENTIALS=...  # реальные данные из Google Search Console
YANDEX_TOKEN=...                    # трафик из Яндекса
TELEGRAM_BOT_TOKEN=...              # Telegram-бот
SPREADSHEET_ID=...                  # выгрузка в Google Sheets
```

Без ключей система работает полностью на моках.  
С ключами подключается реальная аналитика и LLM-кластеризация.
